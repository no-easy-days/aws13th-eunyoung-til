## 인터프리터 언어란?
- 소스 코드를 한 줄씩 해석하면서 바로 실행하는 언어

## vs. 컴파일 언어
| 구분 | 컴파일 언어 (Compiled) | 인터프리터 언어 (Interpreted) | 혼합 구조 / 가상 머신 기반 |
|---|---|---|---|
| 동작 방식 | 소스 코드 → 컴파일 → 실행 파일 → 실행 | 소스 코드 → 인터프리터가 바로 읽고 실행 | 소스 코드 → 중간 코드 → 가상 머신에서 실행 |
| 코드 처리 방식 | 프로그램 전체를 미리 기계어로 변환 | 실행 시점에 코드를 즉석에서 해석 | 중간 코드(bytecode)로 변환 후 실행 |
| 오류 발견 시점 | 실행 전에 오류 발견 | 해당 줄을 실행할 때 오류 발생 | 컴파일 시 + 실행 시 모두 가능 |
| 실행 결과물 | 실행 파일(binary) 생성 | 실행 파일 없음 | 중간 코드 파일 생성 |
| 실행 환경 | OS 및 플랫폼 의존 | 인터프리터 필요 | 가상 머신(JVM 등) 필요 |
| 실행 속도 | 가장 빠름 | 상대적으로 느림 | 중간 수준 (JIT로 개선 가능) |
| 대표 언어 | C, C++, Go, Rust | Python, Ruby, JavaScript | Java, Kotlin, Scala |	

> [!TIP]
> 컴파일 언어는 **실행 성능**을,  
> 인터프리터 언어는 **개발 생산성**을,  
> 가상 머신 기반 언어는 **이식성과 균형**을 중시한다.

## 장점
### 1. 문법이 단순하고 읽기 쉽다
- 영어랑 구조가 비슷함 
- 코드가 짧고 명확 → 생산성 높음

### 2. 개발 속도가 빠르다
- 컴파일 단계 없음 
- 수정 → 실행 → 확인 사이클이 매우 짧음 
<br> → 프로토타이핑, 실험, 자동화에 최적

### 3. 확장성과 결합성이 좋다
- C/C++로 구현된 고성능 라이브러리를 Python에서 쉽게 사용 가능
- 성능이 중요한 부분은 C/C++로 처리하고, 로직과 제어는 Python이 담당
- 외부 언어와의 연동(FFI, 라이브러리 바인딩, embedding)을 고려해 설계된 언어
- 동적 타이핑으로 외부 시스템과의 결합 시 제약이 적음
- 풍부한 라이브러리 생태계로 다양한 기술을 빠르게 조합 가능

→ 고성능 연산과 빠른 개발을 동시에 만족하는 하이브리드 구조에 강함  
  (예: NumPy, TensorFlow, PyTorch)

> [!NOTE]
> **동적 타이핑이란?**
> - 변수를 선언할 때 타입을 명시하지 않음
> - 변수는 타입이 아니라 **값이 타입을 가짐**
> - 같은 변수에 다른 타입의 값 할당 가능

### 4. 많은 라이브러리가 오픈소스로 제공
- 웹: Django, FastAPI
- 데이터: pandas, NumPy
- ML/AI: PyTorch, TensorFlow
- 자동화 / 인프라: boto3, ansible, fabric

### 5. 플랫폼 독립적
- Windows / macOS / Linux 거의 동일하게 실행
- 인터프리터만 있으면 동작

# 단점
### 1. 실행 속도가 느리다
- 인터프리터 기반
- 동적 타이핑으로 인한 런타임 오버헤드

### 2. 런타임 에러 위험
- 타입이 실행 중에 결정됨
- 컴파일 타임에 잡히지 않는 에러 많음

### 3. 모바일 / 클라이언트 앱에 약함
- iOS / Android 네이티브 앱 개발엔 비주류
- 주력 분야는 서버, 백엔드, 자동화, 데이터

### 4. 메모리 사용량이 큼
- 추상화 레벨이 높아 대규모 시스템에서는 비효율적일 수 있음

> [!IMPORTANT]
> Python은 **빠른 실행**보다 **빠른 개발**에 집중한 언어다.
>
> ❌ OS, 게임 엔진, 실시간 시스템  
> ✅ 서버, 클라우드, 자동화, 데이터, AI


> [!NOTE]
> **왜 Python은 느린데도 AI에서 표준일까?**
>
> AI에서는 **코드 실행 속도**보다  
> **고성능 코드를 얼마나 잘 제어하느냐**가 더 중요하다.

### 1. Python은 제어 언어로 사용된다
Python의 역할:
- 모델 구조 정의
- 데이터 파이프라인 구성
- 학습 흐름 제어
- 실험·튜닝·배포 자동화

실제 연산은:
- C / C++
- CUDA (GPU)
- 고성능 수치 라이브러리  
가 담당하며, Python은 지휘자 역할을 한다.

### 2. 핵심 연산은 C/C++로 구현되어 있다
Python 코드 <br>
└── 고수준 API<br>
└── C/C++ 확장 모듈<br>
└── BLAS / CUDA / GPU

> [!IMPORTANT]
> Python은 고수준 인터페이스를 제공하고,  
> 실제 수치 연산과 병렬 처리는 **C/C++ 및 CUDA 레벨**에서 수행된다.

- NumPy: 내부 연산은 C
- PyTorch: C++ + CUDA
- TensorFlow: C++ 코어

### 3. AI에서는 개발 속도가 더 중요하다
- 모델 구조 변경 잦음
- 실험 반복이 핵심
- Python: 수정 → 실행 → 실험이 빠름

> 성능 1% 개선보다 실험을 10번 더 하는 것이 중요

### 4. 수학·데이터 표현력이 뛰어나다
- 아이디어 → 코드 변환이 빠름
- 실험 중심 개발에 유리
